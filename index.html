Part 1 · Decision

Group Analytics - buy it?

A paid add-on (~$2,400/yr) that lets Mixpanel count accounts, not just individual users.

✓ Free without it

  • Break any report down by org_id
  • Active users per account
  • Most of what we need in phase 1

Needs the add-on

  • Count unique accounts (not users)
  • Retention at the account level
  • Funnels where the account converts if anyone in it does
The critical point: we can buy it later and it works retroactively - but only if org_id rides on every event from day one.
Recommendation: don't buy now. The only ask from R&D: org_id on every event - keeps the option open without losing history.
Part 1 · Decision

Auto-capture & user identity

Two data-capture calls to make up front - both hard to undo later.

Auto-capture - Mixpanel logs clicks & pageviews automatically, no code

  • Generic, noisy events ($click) - the "autocapture junk" already in our Lexicontrade-off
  • Can capture text typed into inputs → PII risk on a finance producttrade-off
Strategy: our curated snake_case named events are the source of truth. Keep auto-capture off for the tracking plan; existing autocapture events = migration backlog to deprecate.

User name / email

Store $name / $email on the user profile (people.set after identify) so we can find a specific user in Mixpanel - never as an event or super property (keeps PII out of every event & export). Identity key = stable internal user ID, not the email.
Set up · decision

Group Analytics - do we pay?

Mixpanel counts users by default. The paid add-on lets us count accounts.

What we wantWithout (free)With (paid)
List / rank by org; count active accounts✓ distinct count of organization_id✓
Account retention - do accounts come back✗ engine follows users✓
Account funnels - % of accounts converted✗ converts per user✓
Group profiles - ARR, plan, seats to cohort on✗ none✓
Why we'd pay: letting organization_id be the linking column in retention & funnels is exactly what the add-on does - counting accounts is free; account retention + funnels need it. Group key: organization_account_id (customer) vs organization_id (workspace).

The Mixpanel Kit

3 skills, one pipeline

Measure (why) → Plan (what + Jira) → Publish (activate).  Two human checkpoints: PM sign-off and PR review.

  • mixpanel-measure - decide why before tracking anything.
    A quick guided session that pins down the business question, the metrics that answer it, the North Star, and what "success" means. The antidote to "track every click."
    Output: the Use Cases + Measurement Framework tabs.
  • mixpanel-tracking-plan - turn a feature into a build-ready plan.
    Reads the Figma, the help-center article, and the real product code, then writes the events + properties in the exact 20-column doc R&D needs - North Star event first - and opens the Jira ticket.
    Output: the Tracking Plan (sheet / TSV / CSV) + the R&D ticket.
  • mixpanel-publish - after R&D ships, finish it in Mixpanel.
    Checks the events are actually firing, fills the Lexicon (display names, descriptions, tags, verified) via the Mixpanel MCP, validates data quality (catches junk like "undefined"), then builds the dashboards and metrics.
    Output: enriched Lexicon + validated data + live dashboards.
